Who I am

I am a backend-focused software engineer working primarily with C# and the .NET ecosystem. I enjoy working on systems that are already in production and need to evolve without breaking.

What I focus on

  • Designing maintainable backend architectures
  • Refactoring and stabilizing legacy systems
  • Pragmatic data access patterns beyond CRUD
  • Containerized development and deployment workflows

How I think about software

I believe good software is shaped by constraints: existing users, existing data, and existing code. Most real-world systems are not greenfield, and design decisions must respect reality.

About this blog

This blog is a collection of practical notes, architectural decisions, and real-world examples from backend development. The goal is clarity, not hype.
